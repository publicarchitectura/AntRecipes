Changes in Serialization

The output property standalone="omit" is now recognized. It is equivalent to omitting the property (but is useful if you need to override a value set elsewhere).

The property omit-xml-declaration="yes" is now respected regardless of the encoding, even if this causes the result to be ill-formed XML. This conforms with a change in the specification, made because there is sometimes a need to create fragments of XML that are subsequently concatenated or edited.
